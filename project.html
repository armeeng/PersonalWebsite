  • This code is written entirely in python, and it starts by importing necessary libraries, such as requests, BeautifulSoup, pandas, os, re, math, MinMaxScaler, csv, and openpyxl. These libraries are used for web scraping, data manipulation and analysis, normalization, and file handling.

  • The code defines three lists of URLs, each containing URLs to scrape data from. It then uses the requests and BeautifulSoup libraries to fetch data from the specified URLs. The data is scraped from web pages containing NCAA basketball statistics and placed into a pandas data frame.

  • After web scraping, the code preprocesses the scraped data using pandas. It performs various cleaning tasks to remove unwanted characters and replaces missing values with appropriate defaults. Additionally, it converts certain columns to numerical values for analysis. The data is then normalized using MinMaxScaler from sklearn.preprocessing to bring all the metrics on a common scale. It also defines helper functions for handling errors when accessing values from the dataframes.

  • To compare two NCAA basketball teams, the code iterates through a schedule that contains all the teams that are playing, and creates 3 data frames for each team. These dataframes contain the performance metrics for each team, and the code subtracts the corresponding values between the two opposing teams to identify performance differences which are appended into 3 new data frames. These performance differences can be used to help predict the outcome of a single game, but it still needs quite a bit of work in order to be more accurate and reliable.

  • The code then aims to summarize the differences in performance metrics between the two teams into a singular number. This is done by summing the columns of the comparison dataframes to see which team is statistically weaker. This number, along with the corresponding teams are appended into an excel sheet to keep track of all the games and predictions. However the main purpose of this code is to try and see if there are any statistics that should carry more weight than other statistics (see 'Tracking Statistics After Each Game')

  • After the completion of each game, the code is revisited. This time, when the code is run, the data frames are once again created for each team, but now since the final result of the game are known, we can see which statistics were important in deciding the outcome of the game. Example: if team1 has a higher 3pt shooting percentage than team2, and team1 won the game, we can say that 3pt% has a 1-0 win-loss record. After doing this for quite a few games and quite a few statistics, we get a win percentage for each statistic. The record for these statistics is appended to its own excel sheet and whenever a game is complete the code is ran and updates the excel sheet accordingly. These percentages can then be multiplied with the performance differences to add more weight to the statistics that have more relevance, and less weight to those that are not as important. This creates another comparison data frame which is identical to the previous one except this one is weighted based on the statistics.

  • This description covers most of the project, but there are other pieces not mentioned. There are individual scripts to scrape the web for live/final scores and appended them into the excel sheet. From there a seperate script is used to read the final scores from the excel sheet and assign a value to the winner and loser. Lastly, another script is used to keep track of how well the predictions did, and what games were predicted correctly.

Goals

  • The main objective of this project was to identify the most crucial statistics in determining the outcome of a college basketball game. It is an integral part of a larger project aimed at predicting the winner of college basketball games with precision and reliability, and although this project does 'predict' winners, it does so in a very simple manner and this is something that can definitely be improved. Although basketball games can be influenced by various uncertainties, achieving a higher accuracy than sportsbooks would mark a significant success. While it is challenging to guarantee absolute certainty in predictions, the goal remains to provide more reliable and accurate predictions.

Issues

  • Currently, the main issue with this project is how long it takes to run the code. For a single matchup to be processed, it takes about 15 minutes. During the peak of college basketball season there could be more than 100 different games in a day, which would have a total run time of about 25 hours. There's just not enough time in the day for that. And if I wanted to wait for the games to finish and update the statistical records, I would have to run the code again, which would be another 25 hours.
